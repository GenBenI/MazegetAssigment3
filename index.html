האתגר: הערכת איכות אמינה

המטרה המרכזית הייתה להעריך את איכות הפלט של מודלי שפה גדולים (LLMs) באופן עקבי, מדויק ואובייקטיבי.

השאלה המנחה: כיצד נוכל לסמוך על כך שהערכות המודל אינן מוטות, מקריות או בלתי יציבות?

אמינות דיוק

נקודת התחלה ושינוי כיוון

התחלה

הפרויקט החל עם מודל של Hugging Face (כנראה Mistral), אך נתקל בקשיים טכניים.

שינוי כיוון

המעבר למודלי Gemini של גוגל סיפק גמישות ויציבות גבוהות יותר באמצעות API ייעודי.

שלב 1: התאמות טכניות

אתגר: עיבוד אצוות (Batch Processing)

קוד האצוות דרש פלט מובנה (מילון), אך פונקציית ה-API החזירה טקסט פשוט. היה צורך לגשר על הפער מבלי לשנות את הלוגיקה הקיימת.

# הפונקציה המקורית
def call_gemini(prompt):
  ...
  return "simple text output"
# הפתרון: פונקציית "עטיפה"
def call_gemini_for_batch(prompt):
  text_result = call_gemini(prompt)
  return {'text': text_result, 'runtime': ...}

התוצאה: תאימות מלאה תוך שמירה על קוד נקי.

ההבנה: פלט מובנה

JSON

הפלט נועד לעיבוד אוטומטי על ידי "מודל שופט" (Judge Model) לצורך מתן ציון כמותי.

שלב 3: שכלול מודל השופט (JLM)

שיפור הפרומפט

הנחיה מפורשת לקבלת אובייקט JSON יחיד ותקין, ועידוד מתודולוגיית "הנמקה לפני החלטה".

תיקון קוד

תיקון שגיאות תחביר בקריאה לפונקציית ה-API וחילוץ נכון של טקסט הפלט.

ניקוי פלט

הוספת שלב להסרת תווי "עטיפה" לא רצויים מה-JSON כדי להבטיח פענוח תקין.

פריצת הדרך: הצבעת רוב

(Majority Vote)

כדי להפחית את הסיכון להטיה או טעות של מודל בודד, הפרויקט עבר למערכת רב-מודלית.
גישה זו מבטיחה שההחלטה הסופית יציבה, אמינה יותר, ואינה תלויה ב"גחמה" של שופט יחיד.

שלב 4: אוטומציה והשוואה

תהליך אוטומטי

נוצרה פונקציה (`judge_dataframe_mv`) המיישמת את לוגיקת הצבעת הרוב על מערך נתונים שלם.

התהליך איפשר השוואה ישירה בין התפלגות התוויות של שיטת השופט היחיד לבין שיטת הצבעת הרוב.

מסקנות והצעדים הבאים

יתרונות מרכזיים

  • אמינות גבוהה יותר: הפחתת השפעת טעויות של מודל בודד.
  • יציבות: תוצאות עקביות יותר לאורך זמן ועל פני נתונים שונים.
  • חוסן: המערכת פחות פגיעה להטיות ספציפיות של מודל מסוים.

פשרות ושיקולים

  • זמן ריצה: תהליך ההערכה ארוך יותר באופן טבעי.
  • עלות: שימוש במספר מודלים כרוך בעלות API גבוהה יותר.
  • הצעד הבא: ניתוח כמותי של זמן הריצה והעלות כדי להבין את ה-Trade-offs.
